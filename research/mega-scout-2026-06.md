# Design Book — mega-scout research backlog (2026-06-22)

> 258-agent sweep · 225 research angles · 742 finds → **723 net-new** (not already in the vault). Full machine-readable list: [mega-scout-backlog.json](mega-scout-backlog.json).

## Executive summary

A 13-category sweep (~570 candidate finds) over a very mature zero-dep vault converges on one strategic thesis: the vault's moat is its headless-Chrome SAVE-GATE, and that gate is the single most under-built, least-duplicable, highest-leverage surface remaining. The gate today scores only static-regex slop (hex/duration/radius/left-border-card), an imagery-PRESENCE floor, and a binary "did it draw" — it is blind to accessibility (zero ARIA/contrast-on-gradient/focus-order/reduced-motion-honoring), to typography line-setting (widows/measure/clamp truncation), to overlay/open-state correctness (top-layer/occlusion/clip), to motion budget, to determinism (same-seed=same-pixels), and to WebGPU/context-loss. Almost every category independently nominated a verification gate as its #1 pick. So the plan front-loads ~10 gate capabilities that compound on infrastructure already present (lib/inspect.js already computes WCAG luminance but GIVES UP on gradient backgrounds; render-smoke already reads pixel variance but never checks context-loss/animation-liveness/determinism).

Second thesis: one missing PRIMITIVE unlocks an entire effect class. The shaders runner is single-pass-only (fluid.glsl.js's own header admits it fakes fluid "without ping-pong"), so a ~120-line vanilla PingPong FBO primitive unlocks reaction-diffusion, true fluid, bloom/tonemap chains, and LUT grading — none fakeable today. Likewise a seedable-PRNG + real-CPU-noise pair is the determinism substrate the generative-art skill demands but never shipped.

Third thesis: the platform itself is deleting JS in 2025-26 — CSS anchor positioning, Popover+Invoker, scroll-driven timelines, scroll-state() queries, native carousels, customizable <select>, light-dark(), contrast-color(), field-sizing — and the vault's floating/scroll/theming layers are still 100% getBoundingClientRect/IntersectionObserver/duplicated-token JS. These are net-new, license-free (web standards), and ship behind @supports with the existing JS as the graceful fallback.

Fourth: genuinely NET-NEW capability gaps (not effects) — DTCG token export (the vault's first interchange format; everything is already CSS-var-driven), an accessibility skill (30 skills, zero on a11y), photoreal 3D defaults (scene-runner sets no toneMapping/environment — the #1 flat-metal tell), and runtime imagery (ThumbHash/responsive-<picture> emitter; the vault has skeletons but zero per-image placeholder/srcset). License terrain is clean; the recurring traps to AVOID-as-dep are axe-core (MPL-2.0 — inject unmodified via CDN at test time only, never bundle), LYGIA/paper-design (Prosperity/PolyForm), apca-w3 (restricted — reimplement the formula), and SaaS sources (Mobbin/Gamma/Lovable — inspiration only).

## Cross-cutting trends (2025-26)

- Verification-as-moat: every category independently nominated a save-gate diagnostic as its top pick. The durable competitive surface is rendered-FACT verification (computed style + geometry + pixels sampled under glyphs), NOT pixel-diff baselines — and the vault already runs headless Chrome with half the math in-repo.
- Native CSS is eating JS: anchor positioning, Popover+Invoker Commands, scroll-driven timelines + scroll-state() container queries, native carousels (::scroll-marker/::scroll-button), customizable <select>, light-dark(), contrast-color(), field-sizing, text-box-trim, interpolate-size. All net-new to the vault, license-free, ship behind @supports with the existing JS as the fallback.
- Determinism as a first-class concern (fxhash/Art Blocks 'same hash = same pixels'): seedable PRNG (sfc32) + real CPU noise + a render-twice gate are now table-stakes for generative work and map onto the vault's existing seed-driven compose semantics.
- Stateful/iterative GPU is back: ping-pong FBO reaction-diffusion, neural-CA, compute particles — the 'alive texture' looks the single-pass shader era skipped. The PingPong primitive is the gate to all of it.
- WebGPU/TSL migration is production (Chrome/FF/Safari late-2025) but compute() is WebGPU-only and does NOT fall back — a real WebGPU→WebGL2-transform-feedback→CPU ladder plus WebGPU-aware verification is mandatory, not optional.
- Anti-AI-slop / tactile-print reaction: risograph overprint, CSS halftone, dithering, deckle edges, neo-brutalism, lit-emboss paper texture, true baseline grids — physical/human surfaces that directly counter the 'gradients + no images = AI tell' authenticity law.
- Perceptual color by construction: APCA/reverse-contrast solvers compose to HIT a target Lc instead of checking after; OKLCH relative-color derives whole token systems from one hue; P3 wide-gamut as progressive enhancement. (License trap: apca-w3 is restricted — reimplement the formula.)
- Accessibility graduated from advice to enforceable baseline: WCAG 2.2 (ISO Oct 2025) added mechanically-verifiable target-size/dragging/focus-not-obscured; forced-colors replaced -ms-high-contrast; APG behaviors are expected. The vault is thin here.
- Compact-placeholder + Core-Web-Vitals discipline matured: ThumbHash/CSS-LQIP + explicit width/height + fetchpriority + AVIF/WebP <picture> are now codifiable as a gate dimension, not just authoring advice.
- DTCG 2025.10 is the first real token-interchange standard (Figma/Penpot/Style Dictionary/Tokens Studio read it) — and a token-driven vault is the perfect producer. Note the breaking change: color $value is now a structured object, not flat hex.

## Remaining capability gaps

Even after this plan, Design Book retains notable capability gaps. INFRASTRUCTURE: no FBO/multi-pass primitive yet (PingPong is the #1 fix, but until it lands the entire stateful-shader class is impossible); no seedable-PRNG/CPU-noise substrate (generative-art skill demands it, ships nothing); no auto-layout engine at all (no Sugiyama/dagre/ELK) so diagrams can only fill fixed templates, never lay out an arbitrary graph; no machine-readable param-manifest so the composer cannot enumerate/drive generative knobs headlessly (Tweakpane is human-only). 3D: scene-runner is WebGLRenderer-only with no WebGPU/TSL sibling, no orbit/damping/fit camera, no GLTFLoader with Draco/KTX2 decoders (compressed .glb fails to load entirely), no Gaussian-splat support, no vanilla physics, no point-cloud/LiDAR path, no postprocessing stack beyond single bloom. ICONS: the vault has NO icon-rendering engine — icon-picker is emoji chars; there is no resolver for the 200k+ permissive icon corpus. VERIFICATION still-missing: no full-text search engine (palette is static-array fuzzy); no print/PDF page-fit or email-compat (caniemail) gate; no JSON-LD/structured-data emit-or-validate; no internal-link/dead-anchor resolution; no motion-budget/effect-stacking gate; no GPU-frametime budget probe. A11y depth: no dismiss-stack manager (nested overlays all collapse on one Escape), no type-ahead listbox search, no focus-appearance pixel gate. ARCHETYPE breadth: ~15+ uncomposable page genres (api-reference, email-newsletter, changelog page, glossary, comparison, conference-landing, resume, scrollytelling-explainer, link-in-bio, case-study) and no box/wireframe-spec front-door (composer is genre-driven only — no region-IR input). DATAVIZ: missing whole FT-taxonomy buckets (boxplot, violin, diverging-bar, bullet, chord, beeswarm), no canvas/streaming data path, no real map engine or projection (only a hardcoded Greece bake), no big-data downsampling (LTTB). FAN-OUT: book_variants re-runs seeds with NO measured pairwise distinctness — variants can be cosmetic reshuffles.

## Per-category top picks

### WebGL / GPU shader effects (38 finds)
_The 2024-2026 shader-fx wave is dominated by (1) retro/print stylization — ordered Bayer + blue-noise dithering, CRT scanline/aperture-grille/barrel passes, and NPR "post-processing as a medium" (luminance cross-hatch, Kuwahara oil-paint, edge-detect ASCII), all popularized by Maxime Heckel's 2024-25 series and Codrops; (2) iOS-26 "Liquid Glass" — true background/lens refraction via SVG feDisplacementMap as backdrop-filter (Chromium-only) with WebGL screen-space-refraction upgrades; and (3) the WebGPU + TSL transition, where compute-shader GPGPU particle fields (1M+ points) are the new ceiling, with the critical gotcha that TSL/WebGPURenderer auto-falls-back only the RENDER path to WebGL2 — compute itself is WebGPU-only, so a real fallback ladder (WebGPU compute -> WebGL2 transform-feedback -> CPU) is mandatory. Cross-cutting all of this: the vault's single-pass screen-only runner.js is the bottleneck — almost every high-value effect (reaction-diffusion, true fluid, bloom/tonemap/DoF, LUT grade) actually needs an FBO ping-pong primitive the vault lacks, making that primitive the highest-leverage enabling add. License hygiene is the recurring trap: LYGIA (Prosperity), paper-design/shaders (PolyForm), particlesGL (paid-commercial), and most cinematic LUT/.cube packs (redistribution-forbidden or CC-BY-SA) are inspiration-only; the clean path is vanilla reimplementation from public-domain math (Bayer recurrence, Blinn metaballs, Kuwahara 1976, Lottes CRT public-domain) plus CC0 assets (blue-noise texture, self-baked HaldCLUTs) and confirmed-MIT references (three.js, dmnsgn/glsl-tone-map, GarrettGunnell, tdhooper). The emerging moat is verification: context-loss survival (WEBGL_lose_context), GPU frametime budgets (EXT_disjoint_timer_query), and NaN/black-frame correctness gates — none of which any competing design tool ships._
